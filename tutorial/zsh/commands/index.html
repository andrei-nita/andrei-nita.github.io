Zsh

Commands

Linux comes with useful commands that run from the terminal. To check where the commands are stored on your OS, type at the terminal:

which date
# => /usr/bin/date

/usr/bin/ is the location, on Manjaro OS, of all the commands that are showed in the following pages.

Commands can also run in the background if an ampersand & is placed after a command. A background commands output is printed to the terminal, but it cannot read from the input.

Run command in the background

sleep 30 &

Display command that runs in the background.

jobs
# => [1]+  Running  sleep 30 &

The command sleep is executed in the background, and after 30 seconds the command will end.

Bring the job to the foreground

fg
# => [1]  + 98656 running    sleep 30

fg brings to the foreground the sleep process, if 30s have not passed, and waits for sleep to finish.

Kill or pause the process

Use Ctrl-C to kill the process, or Ctrl-Z to pause it

If Ctrl-C is used after fg, the sleep process will end even if the 30s have not passed.

If Ctrl-Z is used, then the sleep process will pause until it is resumed.

Resume command

# Resume a background process after it has been paused with Ctrl-Z
bg

With bg the sleep process will resume and end when the 30s have passed.
