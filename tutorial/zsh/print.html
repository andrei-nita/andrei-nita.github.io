Zsh

★★★☆☆ These notes have enough examples to make the topic clear.

Introduction

Zsh is a shell available on Unix systems. A shell is a CLI (command line interpreter) and, prior to 1970, was the only user interface available for computers; there were no GUI.

Many Linux distributions come with Bash as the default shell. The command listed on this pages work for both Bash and Zsh; as they are based on Bourne Shell.

Definitions

  • Zsh: a shell available for Unix systems.
  • Shell: it's a program that takes commands from a terminal and gives them to the Kernel (low-level part of an Operating System).
  • Terminal: program that opens a graphical window from where the user can interact with the shell.

Why use zsh:

Here are some of the benefits over Bash:

  • Plugin and theme support through which you can achieve better productivity.
  • Automatic cd: just type the name of the directory
  • Spelling corrections

Operating Systems

If you have a Linux installed, then you are ready to go. If instead you are using Windows and want to use Zsh or Bash because you need it for development tasks, then here are some options:

  • install Cygwig
  • install WSL RECOMMENDED

Installation

1. Check zsh version

To find if zsh is installed on your system, type at the terminal the fallowing command:

zsh --version

Output: zsh 5.9 (x86_64-pc-linux-gnu) or similar.

If you don't have zsh installed jump to step 3.

2. Check if zsh is the default shell

echo $SHELL

Output: /usr/bin/zsh or similar

If zsh is not activated, jump to step 4.

3. Install Zsh

Ubuntu

sudo apt install zsh

Manjaro

pacman -S zsh

Windows WSL

apt install zsh

Mac

  • With brew:
brew install zsh
  • For recent versions of Mac:
chsh -s /usr/local/bin/zsh
  • For macOS High Sierra and older:
chsh -s /bin/zsh

For examples of installations for other OS, check this guide.

4. Make zsh the default shell

chsh -s $(which zsh)

Hello World

echo is the command used to display a line of text. It can be useful for scripts to create the user interface.

Simple example

echo Hello world! 
# => Hello world!

In the example above, the first line is the command passed to the terminal. echo represent the command and Hello world! represent the parameter passed to the command.

The second line is a comment. The # character indicates that the line should be treated by the shell like a comment and everything written on the line after the hash mark will be ignored. Comments are very useful for documentation.

So, in the example above, on the second line, I used a comment # => Hello world! to indicate the output of the first line.

Script example

Create a file on the desktop, for example, echo.sh and paste the following:

#!/usr/bin/env bash

echo Hello world!

On linux, like Ubuntu, right-click the desktop and choose Open in terminal from the menu. At the terminal type:

./echo.sh

The difference from the other example is that when running a script we have to insert a shebang.

Precisely, the first line of the script is the shebang which tells the system how to execute the script. Note that shebang is also a comment.

More info

man echo

       echo [SHORT-OPTION]... [STRING]...

Variables

Declare variable

Variable="Some string"

Variable="Some string" must be written without spaces otherwise bash will trigger an error because will think that the variable is a command instead of a variable declaration."

Use variable

echo $Variable # => Some string
echo "$Variable" # => Some string
echo '$Variable' # => $Variable

Parameter expansion

echo ${Variable} # => Some string

Parameter expansion is created with ${ } and this syntax means to expand the value. During the expansion time, the value or parameter can be modified.

Examples:

  • String substitution in variables

echo ${Variable}        # => Some string
echo ${Variable/Some/A} # => A string
# The first occurrence of "Some" is substitute with "A"

  • Substring from a variable

Length=7
echo ${Variable:0:Length} # => Some st
# This will return only the first 7 characters of the value
echo ${Variable: -5} # => tring
# This will return the last 5 characters (note the space before -5)

  • String length

# String length
echo ${#Variable} # => 11

  • Indirect expansion

OtherVariable="Variable"
echo ${!OtherVariable} # => Some String
# This will expand the value of OtherVariable

  • Default value

echo ${Foo:-"DefaultValueIfFooIsMissingOrEmpty"}
# => DefaultValueIfFooIsMissingOrEmpty
# This works for null (Foo=) and empty string (Foo=""); zero (Foo=0) returns 0.
# Note that it only returns default value and doesn't change variable value.

Built-in Variables

Built-in variables are variables provided by the shell. Here are some useful build-in variables:

$?

echo "Last program's return value: $?"

$$

echo "Script's PID: $$"

$#

echo "Number of arguments passed to script: $#"

$@

echo "All arguments passed to script: $@"

$1 $2...

echo "Script's arguments separated into different variables: $1 $2..."

More than 9 arguments are also possible by using braces, e.g. ${10}, ${11}, ...

pwd

# Our current directory is available through the command `pwd`.
# `pwd` stands for "print working directory".
# We can also use the built-in variable `$PWD`.
# Observe that the following are equivalent:
echo "I'm in $(pwd)" # execs `pwd` and interpolates output
echo "I'm in $PWD" # interpolates the variable

Arrays

Arrays store multiple values.

Declare an array

name="zsh"
# Declare an array with 6 elements
array0=(name two three four five six)

Print first element

echo $array0 # => "zsh"
# or
echo ${array0[0]} # => "zsh"

Save element to variable

firstValue=$array0
echo ${firstValue} # => "zsh"
# or
firstValue=${array0[0]}
echo ${firstValue} # => "zsh"

Print all elements

echo ${array0[@]} # => "one two three four five six"
# or
for i in "${array0[@]}"; do
    echo "$i"
done

Print number of elements

echo ${#array0[@]} # => "6"

Print number of characters in third element

echo ${#array0[2]} # => "5"

Print range of elements

# Print 2 elements starting from forth
echo ${array0[@]:3:2} # => "four five"

Loops

Loops iterate over the arguments given or until a condition is accomplished.

for

for num in {1..3}
do
    echo "$num"
done
# => 1
# => 2
# => 3

for loop, in the example above, iterate over the range from 1 to 3 saving the value to the variable num which gets printed at each iteration.

for old syntax

for ((a=1; a <= 3; a++))
do
    echo $a
done
# => 1
# => 2
# => 3

for loop over files

Create a file loop.sh and paste the following:

#!/usr/bin/env bash

echo "Hello from file1" > file1 #create file1
echo "Hello from file2" > file2 #create file2

# They can also be used to act on files..
# This will run the command `cat` on file1 and file2
for Variable in file1 file2
do
    cat "$Variable"
done

# => Hello from file1
# => Hello from file2

Execute the file with ./loop.sh.

for loop over command

# ..or the output from a command
# This will `cat` the output from `ls`.
for Output in $(ls)
do
    cat "$Output"
done

for loop over pattern

# Bash can also accept patterns, like this to `cat`
# all the Markdown files in current directory
for Output in ./*.markdown
do
    cat "$Output"
done

while loop

variable=true

# while loop:
while [ $variable ]
do
    echo "loop body here..."
    break
done
# => loop body here...

In the example above, the while loop iterates until $variable became false or a break is used.

Logic

if

# use `man test` for more info about conditionals
if [ $Name != $USER ]
then
    echo "Your name isn't your username"
else
    echo "Your name is your username"
fi
# True if the value of $Name is not equal to the current user's login username

Note

# if $Name is empty, bash sees the above condition as:
if [ != $USER ]
# which is invalid syntax
# so the "safe" way to use potentially empty variables in bash is:
if [ "$Name" != $USER ] ...
# which, when $Name is empty, is seen by bash as:
if [ "" != $USER ] ...
# which works as expected

if with and, or

# To use && and || with if statements, you need multiple pairs of square brackets:
if [ "$Name" == "Steve" ] && [ "$Age" -eq 15 ]
then
    echo "This will run if $Name is Steve AND $Age is 15."
fi

if [ "$Name" == "Daniya" ] || [ "$Name" == "Zach" ]
then
    echo "This will run if $Name is Daniya OR Zach."
fi

Conditional execution

# There is also conditional execution
echo "Always executed" || echo "Only executed if first command fails"
# => Always executed
echo "Always executed" && echo "Only executed if first command does NOT fail"
# => Always executed
# => Only executed if first command does NOT fail

if with regex

# There is also the `=~` operator, which tests a string against a Regex pattern:
Email=me@example.com
if [[ "$Email" =~ [a-z]+@[a-z]{2,}\.(com|net|org) ]]
then
    echo "Valid email!"
fi
# Note that =~ only works within double [[ ]] square brackets,
# which are subtly different from single [ ].
# See https://www.gnu.org/software/bash/manual/bashref.html#Conditional-Constructs for more on this.

case

# Bash uses a `case` statement that works similarly to switch in Java and C++:
case "$Variable" in
    # List patterns for the conditions you want to meet
    0) echo "There is a zero.";;
    1) echo "There is a one.";;
    *) echo "It is not null.";;  # match everything
esac

Retrive retrun value

foo arg1 arg2

resultValue=$?

# => And: arg1 arg2...

Redirection and Pipe

Redirection is a technique used to pass output to a file.

Pipe is a technique used to pass output to another program.

Redirections

output > operator

date > date.txt

In the example above the date command writes his output to the date.txt file. Running the command another time will override the contents of the file.

append >> operator

date >> date.txt

Running the command above will append the output to the content of the date.txt file.

input < operator

Copy and paste at the terminal the following:

cat > sort.txt << EOL
b
c
a
EOL

Then redirect the output of the file as input to sort.

sort < sort.txt
# => a
# => b
# => c

error 2> operator

aaa # command not found

aaa 2> /dev/null

In the last example the error is redirected to the /dev/null so no error is displayed on the screen.

Pipe operator

ls /etc | less

| takes the output of ls and pass it to less

Press q to exit from less

Other Features

Substitution

# Commands can be substituted within other commands using $( ):
# The following command displays the number of files and directories in the
# current directory.
echo "There are $(ls | wc -l) items here."

# The same can be done using backticks `` but they can't be nested -
# the preferred way is to use $( ).
echo "There are `ls | wc -l` items here."

Expressions

# Expressions are denoted with the following format:
echo $(( 10 + 5 )) # => 15

Aliases

# Redefine command `ping` as alias to send only 5 packets
alias ping='ping -c 5'

# type only c when clearing the screen

Commands

Linux comes with useful commands that run from the terminal. To check where the commands are stored on your OS, type at the terminal:

which date
# => /usr/bin/date

/usr/bin/ is the location, on Manjaro OS, of all the commands that are showed in the following pages.

Commands can also run in the background if an ampersand & is placed after a command. A background commands output is printed to the terminal, but it cannot read from the input.

Run command in the background

sleep 30 &

Display command that runs in the background.

jobs
# => [1]+  Running  sleep 30 &

The command sleep is executed in the background, and after 30 seconds the command will end.

Bring the job to the foreground

fg
# => [1]  + 98656 running    sleep 30

fg brings to the foreground the sleep process, if 30s have not passed, and waits for sleep to finish.

Kill or pause the process

Use Ctrl-C to kill the process, or Ctrl-Z to pause it

If Ctrl-C is used after fg, the sleep process will end even if the 30s have not passed.

If Ctrl-Z is used, then the sleep process will pause until it is resumed.

Resume command

# Resume a background process after it has been paused with Ctrl-Z
bg

With bg the sleep process will resume and end when the 30s have passed.

sudo

sudo allows a permitted user to execute a command as the superuser or another user

sudo whoami
# => root

After inserting the password whoami display the sudo user which is root.

sudo automatic password

sudo -S

-S flag will read the password from standard input. This is useful, for example, for a script that has to run commands with sudo without the user typing the password at the terminal.

echo "your_password" | sudo -S whoami

here string

Another way of passing the password to sudo is using a here string <<<.

sudo -S <<< "228086" whoami

Create user

# add new user
sudo adduser <user_name>

# add administrative rights
sudo usermod -aG sudo <user_name>

Back

ls

ls command lists information about the FILEs (the current directory by default). Sort entries alphabetically if none of -cftuvSUX nor --sort is specified.

l # List all files, also the hidden ones.
ls # Lists the files and subdirectories contained in the current directory
ls -l # Lists every file and directory on a separate line
ls -t # Sorts the directory contents by last-modified date (descending)
ls -R # Recursively `ls` this directory and all of its subdirectories
ls -l | grep "\.txt" # The `grep` command filters the input with provided patterns.

For more information check man ls.

cp

cp copies files or directories from one place to another. cp creates NEW versions of the sources, so editing the copy won't affect the original (and vice versa).

Note that it will overwrite the destination if it already exists.


cp srcFile.txt clone.txt
cp -r srcDirectory/ dst/ # recursively copy

Look into rsync if you plan on exchanging files between computers.

rsync

rsync is a remote and local file-copying tool.

rsync is used for backups and mirroring and as an improved copy command for everyday use.

Not on every system rsync is installed. For example, for Ubuntu you need to install it: sudo apt install rsync

Useful flags

-a for preserve the metadata such as permissions, time, symbolic links etc.
-v display verbose output
-r copy recursively all sub-directories and its contents
-z compress the file
-P show progres bar
-e remote shell to use

simple transfer

rsync -ar <LOCAL_DIR> <USER>@<IP>:<REMOTE_DIR>

transfer

rsync -arvz -P -e 'ssh -p 22' <LOCAL_DIR> <USER>@<IP>:<REMOTE_DIR>

exclude

rsync -arvz -P -e 'ssh -p 22' --exclude '.*' --exclude '*.go' --exclude-from '/path/file_specify_what_to_exclude.txt' path/to/transfer <USER>@<IP>:<REMOTE_DIR>

ssh

ssh (SSH client) is a program for logging into a remote machine and for executing commands on a remote machine. It is intended to provide secure encrypted communications between two untrusted hosts over an insecure network.

Install ssh key

ssh-keygen

Transfer ssh key to server

The command below will scan the local account for the id_rsa.pub key and copy the contents of the ~/.ssh/id_rsa.pub key into ~/.ssh/authorized_keys remote file.

ssh-copy-id -p 22 <USER>@<IP>

Further connections to the remote server will not ask anymore for the password.

rm removes each specified file. By default, it does not remove directories.

Note: removed commands cannot be undone

Remove file

Remove files

chmod

chmod changes the file mode bits of each given file according to mode.

Numeric mode

# Allow to all (user, group, others) read permissions
chmod 444 file

In the numeric mode the permissions are as fallow:

read = 4
write = 3
execute= 1

Symbolic mode

The format of a symbolic mode is ugoa +-=.

# allow read permission to everyone
chmod a+r file

Here is what the letters mean:

u stands for user.
g stands for group.
o stands for others.
a stands for all.

r: read
w: write
x: execute

+: add permission
-: remove permission
=: make the only permissionthe file has

Options

Options
  -f, --silent, --quiet   Suppress most error messages.

  -v, --verbose           Output a diagnostic for every file processed.
  -c, --changes           like verbose but report only when a change is made.

      --reference=RFile   use RFile's mode instead of MODE values.

  -R, --recursive         Change files and directories recursively.
                          Take care to not run recursive chmod on the root '/' directory or any other system directory.

      --help              Display help and exit.

      --version           output version information and exit.

grep

grep searches for PATTERNS in each FILE. PATTERNS are one or more patterns separated by newline characters, and grep prints each line that matches a pattern. Typically, PATTERNS should be quoted when grep is used in a shell command.

Lines that match Regex

# print to stdout all lines of file.txt which match some regex
# The example prints lines which begin with "foo" and end in "bar"
grep "^foo.*bar$" file.txt

Number of lines

# pass the option "-c" to instead print the number of lines matching the regex
grep -c "^foo.*bar$" file.txt

Scan recursively

grep -r "^foo.*bar$" someDir/ # recursively `grep`
# or
grep -rI "^foo.*bar$" someDir/ # recursively `grep`, but ignore binary files

Line numbers

grep -n "^foo.*bar$" file.txt # give line numbers

Filter

# perform the same initial search, but filter out the lines containing "baz"
grep "^foo.*bar$" file.txt | grep -v "baz"

Search for string

# if you literally want to search for the string,
# and not the regex, use `fgrep` (or `grep -F`)
fgrep "foobar" file.txt

less

less is a program similar to more(1), but which allows backward movement in the file as well as forward movement. Also, less does not have to read the entire input file before starting, so with large input files it starts up faster than text editors.

ps aux | less
# view the output of the ps command page by page

The navigation commands are:

Down arrow, Enter, e, or j       Move forward one line.

b	                         Move Backward one page.
/pattern	                 Search forward for matching patterns.
?pattern	                 Search backward for matching patterns.
n	                         Repeat previous search.
N	                         Repeat previous search in reverse direction.
g	                         Go to the first line in the file.
Ng	                         Go to the N-th line in the file.
G	                         Go to the last line in the file.
p	                         Go to the beginning of fthe ile.
Np	                         Go to N percent into file.
h	                         Display help.
q	                         Exit less.
